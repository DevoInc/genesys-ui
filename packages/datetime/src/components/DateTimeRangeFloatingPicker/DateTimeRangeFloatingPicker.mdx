# DateTimeRangeFloatingPicker

Component used for selecting a range of date-time values. The component provides options
for specifying the format of the range, including real-time control and shift modifiers.
The calendar and preset selections are rendered in a floating panel.

The `DateTimeRangeFloatingPicker` component is an combination of the `DateTimeRange`
component and `DateTimeRangeInput` component.

Users can manipulate the value either directly from the `DateTimeRange` component
or via `DateTimeRangeInput` component.

Each time the inputs text field value are modified, it undergoes validation
against accepted date formats.
The first value in the `dateFormats` array serves as the default value to display.
If no format is specified, the default value will be `YYYY-mm-dd HH:mm:ss`.

When the `Enter` key is pressed and `DateTimeRange` componets has de focus
the `onChange` method of the component is triggered.

Any change made to the `DateTimeRange` component triggers the `onChange` method.

## parseDate & formatDate

If we need a custom time language we can use the properties `parseDate` &
`formatDate`.

- `parseDate`: Allow us to define the parser of the language, for example, our
  language could accept strings like `now() - 5m`, this is the place for parse
  this string and validate it, etc.
- `formatDate`: Allow us to visualize our date in the form of the custom
  language if needed

<Canvas of={Stories.CustomTimeLanguage} />

## Limit the valid dates with parseRange and parseDate

- `parseDate` also allow to check if a day is valid in the calendar and set a
  tooltip with the reason if not.
- `parseRange` is the same as `parseDate` but for the range the main and default
  behavior is to avoid the user insert an inverted range.

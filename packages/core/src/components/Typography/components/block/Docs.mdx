The block typographic components are used inside another components or as a
base of them: `HelperMessage`, `ValidationMessage`, `FieldLabel`... etc.

## BlockQuote

`Typography.BlockQuote` renders an HTML Block Quotation Element: indicates that
the enclosed text is an extended quotation. A URL for the source of the quotation
may be given using the 'cite' attribute.

## List and ListItem

`Typography.List` can render an HTML `<ul>` (by default) or `<ol>` which represents an
unordered/ordered list of items, typically rendered as a bulleted list.
Each child has to be a `Typography.ListItem` component, which renders an
HTML `<li>` element.

## Heading

`Typography.Heading` generates HTML headings which are the titles or subtitles
of our components: Panel Header titles, BoxMessage titles, Grouping titles
in Menus and forms... etc.

If we want the standard HTML headings (h1-h6) we have to use any 'h-' value for
type prop. They render an HTML `div` by default, but it's possible to use any
`h1` to `h6`, using the 'as' prop.

### [Basic usage](#basic-usage)

<Canvas of={HeadingStories.Heading} />

If we need extra-big headings for commercial or on boarding contexts, we can use
any 'hero-' value for 'type' prop:

<Source
  code={`
<Typography.Heading type={'hero-md'} >Heading</ Typography.Heading>
`}
/>

If we need a subtle heading to group menu items, form fields... etc. we can
use any 'overline-' value for 'type' prop:

<Source
  code={`
<Typography.Heading type={'overline-md'} >Heading</ Typography.Heading>
`}
/>
